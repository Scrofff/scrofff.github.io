DONE Spend some more time with fswatch

  • fswatch is a utility which can watch files and react to them changing state.
  • I want to watch the files which are being worked on and react when they are updated – I don't need to know which file has changed, just that one or more of them has. The following invocation of fswatch allows this.
fswatch --event Updated -o ~/Org/Roam
  • Pipe the output from this command into a bash script to do whatever you want to do when the change has occurred. For example:
while read NUM ;
do
  pdflatex test
done
  • In my use-case, I want to export the master file when any of the constituents have changed. The following use of emacsclient performs the export on the desired file.
emacsclient -e "(progn (find-file \"~/Test/test.org\") (org-export-and-save) (previous-buffer))"
  • A complete command line which looks for updated files, exports the master, and runs LaTeX on it is given below.
fswatch --event Updated -o ~/Org/Roam/ | while read NUM ; do emacsclient -e "(progn (find-file \"~/Test/test.org\") (org-export-and-save) (previous-buffer))" ; pdflatex -synctex=1 test ; done

PROG Sharing emacs configurations

  • Syncthing doesn't work on two local directories - well, it could but you'd have to run two instances of the daemon on different ports. More trouble than it's worth.
  • Dear old rsync and Dropbox seems to be the way. Dropbox to share between machines and installations, rsync to synchronise the directories using a command such as:
rsync -arv --progress --delete ~/.emacs.d/ /mnt/h/Dropbox/Emacs/emacs.d
